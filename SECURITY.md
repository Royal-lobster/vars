# Security Policy

## Supported Versions

Security fixes are generally applied to the latest development state of the project.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public GitHub issues.

If private vulnerability reporting is enabled for this repository, use GitHub's private reporting flow. Otherwise, contact the maintainer privately through GitHub and include:

- A description of the issue
- Steps to reproduce it
- Impact and affected components
- Any suggested mitigation, if you have one

You can expect an initial acknowledgment within a reasonable time, followed by triage and a coordinated fix if the report is valid.

## Scope

This project handles encrypted configuration and secret material, so reports involving key handling, encryption, secret disclosure, CLI export behavior, or unsafe defaults are especially helpful.

For general hardening ideas that are not sensitive, open a normal GitHub issue instead.
