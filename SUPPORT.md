# Support

## Getting Help

- Read `README.md` first for the project overview and quick start.
- Check the docs site for usage details and CLI reference.
- Search existing GitHub issues before opening a new one.

## Where To Ask

- Bug reports: open a GitHub issue with reproduction steps
- Feature requests: open a GitHub issue describing the use case
- Security issues: follow `SECURITY.md` and do not post details publicly

## What To Include

When opening an issue, include as much of the following as possible:

- Your operating system and Node.js version
- The command you ran
- The expected result
- The actual result
- A minimal reproducible example
