## Install

```bash
npx vars init
```

You'll be asked for a PIN. After that, you get:
- `config.vars` — your variables file, encrypted at rest and safe to commit
- `.vars/key` — your encryption key, locked behind that PIN (gitignored)
- `vars.generated.ts` — typed accessors, auto-regenerated
- A `.gitignore` update so the key doesn't get committed
- A pre-commit hook that blocks commits if secrets are still decrypted

Already have a `.env`? vars will offer to import it during init.

## Add a variable

```bash
npx vars show config.vars
```

This decrypts `config.vars` in place so you can edit it. Open the file and add a variable:

```vars
DATABASE_URL : z.string().url() {
  dev  = postgres://localhost/myapp
  prod = postgres://prod-host/myapp
}
```

The schema (`z.string().url()`) sits right next to the name. vars validates against it, so a typo in the connection string gets caught before it reaches your app.

Need to annotate a variable with extra metadata? Use parenthesized metadata after the schema:

```vars
DATABASE_URL : z.string().url() (description = "Primary database connection string") {
  dev  = postgres://localhost/myapp
  prod = postgres://prod-host/myapp
}
```

Non-secret values that don't need encryption can be marked with the `public` keyword:

```vars
public APP_NAME : z.string() {
  dev  = myapp-dev
  prod = myapp
}
```

## Encrypt

```bash
npx vars hide
```

Encrypts all secret values with AES-256-GCM. Safe to commit.

## Run your app

```bash
npx vars run --env dev -- node server.js
```

vars decrypts in memory and injects the values as environment variables before handing off to your process. Nothing gets written to disk.

## Generate typed accessors

```bash
npx vars gen config.vars
```

Outputs `vars.generated.ts` with a typed export for each variable. No more guessing whether `process.env.DATABASE_URL` exists. Public variables get plain types; secrets get a `Redacted` wrapper so you don't accidentally log them:

```ts
// vars.generated.ts (auto-generated, do not edit)
export const env = {
  APP_NAME: string,          // public — plain value
  DATABASE_URL: Redacted<string>, // secret — unwrap explicitly to access
};
```

This file is also auto-regenerated by `vars show` and `vars hide`.
