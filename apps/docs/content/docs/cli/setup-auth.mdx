## `vars init`

```bash
vars init
```

Sets up a new vars project. vars:

1. Creates `config.vars` â€” single encrypted secrets file (safe to commit)
2. Creates `.vars/key` â€” master encryption key (gitignored)
3. Updates `.gitignore` â€” key and swap files excluded
4. Adds `#vars` import to `package.json` scripts

If a `.env` file exists, vars offers to import it. It reads the key-value pairs, guesses Zod schemas from the values, and encrypts everything before writing.

If `config.vars` already exists but `.vars/key` is missing (e.g., a fresh clone), running `init` again detects the incomplete setup and creates the missing key file.

### Flags

| Flag | Default | What it does |
|------|---------|-------------|
| `--file`, `-f` | `.env` | Path to the `.env` file to import |
| `--env` | `dev` | Environment name for imported values |

### Example

```bash
# Import from a non-default path, label values as "staging"
vars init --file .env.staging --env staging
```

## `vars key`

Subcommands for managing your encryption key.

### `vars key init`

```bash
vars key init
```

Generates a new `.vars/key` file. Run this on a fresh clone that already has a `config.vars` committed.

### `vars key fingerprint`

```bash
vars key fingerprint
```

Prints a short fingerprint of the current key. Use it to verify that two machines share the same key without exposing the key itself.

### `vars key export`

```bash
vars key export
```

Prints the base64-encoded master key to stdout. Requires the master PIN â€” owner PINs cannot export. Pipe it to a secure channel when sharing with a teammate or storing in CI.

```bash
# Copy key to clipboard (macOS)
vars key export | pbcopy
```

## `vars pin`

Subcommands for managing owner-scoped PINs. See [Multi-PIN](/docs/security/multi-pin) for the full explanation.

### `vars pin create`

```bash
vars pin create <owner>
```

Creates a scoped PIN for an owner. Requires the master PIN. The CLI derives a sub-key from the master key, wraps it with the new PIN, and re-encrypts all fields tagged with that owner.

```bash
# Create a PIN for the backend team
vars pin create backend-team

# The backend team can now use their PIN with show/hide
vars show   # enter backend-team PIN â†’ only their fields decrypt
```

Owner names may only contain letters, digits, hyphens, and underscores.

### Revoking an owner PIN

There is no `vars pin delete` command. To revoke an owner's access, remove their `owner=<name>` line from `.vars/key` and redistribute the key file. The key file is plain text â€” open it in any editor.

## `vars doctor`

```bash
vars doctor
```

Runs health checks and prints a summary. Exits 1 if any check fails.

### Checks

Files:
- `config.vars` exists
- `.vars/key` exists
- Codegen is up to date (generated file matches current schemas)

Security:
- `.vars/key` is gitignored
- Pre-commit hook installed
- All values are encrypted

Secrets Health:
- No expiring or deprecated secrets (warns if `@expires` is within 30 days, or past due)

## `vars completions`

```bash
vars completions <shell>
```

Writes a shell completion script to stdout. Redirect it to wherever your shell loads completions from.

### Supported shells

| Shell | Setup |
|-------|-------|
| `bash` | `vars completions bash >> ~/.bash_completion` |
| `zsh` | `vars completions zsh > ~/.zsh/completions/_vars` |
| `fish` | `vars completions fish > ~/.config/fish/completions/vars.fish` |

Open a new shell after running and tab-completion works for all vars subcommands.
