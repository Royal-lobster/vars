## template

```bash
vars template --env prod > .env.production
```

Decrypts your vault for a given environment and writes a valid `.env` to stdout. Pipe it wherever.

stdout gets the `.env`. stderr gets status messages. Pipes work fine.

Values with spaces, `#`, or `=` get quoted automatically. Encrypted values are skipped in non-interactive mode (CI, pipes) so you don't get a passphrase prompt halfway through a pipeline.

**Flags**

| Flag | Short | Description |
|------|-------|-------------|
| `--env <name>` | | Environment to render (default: `dev`) |
| `--file <path>` | `-f` | Use a different vault file |

**Examples**

```bash
# Write a .env file for prod
vars template --env prod > .env.production

# Pipe straight into your deploy script
vars template --env staging | some-deploy-tool --env-file /dev/stdin

# Use a non-default vault
vars template --env prod -f .vars/backend.vars > backend.env
```

## pull <span style={{ color: "var(--color-fd-muted-foreground)", fontSize: "0.75em", fontWeight: "normal", marginLeft: "0.5em" }}>coming soon</span>


Pulls environment variables from your platform, encrypts them, and merges them into your `.vars` vault. Existing values aren't overwritten by default.
