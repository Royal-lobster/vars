## Install

```bash
npx vars init
```

You'll be asked for a PIN. After that, you get:
- `.vars/vault.vars` — your variables file, encrypted at rest
- `.vars/key` — your encryption key, locked behind that PIN (gitignored)
- `.vars/vars.generated.ts` — typed accessors, auto-regenerated
- A `.gitignore` update so the key and unlocked files don't get committed
- A pre-commit hook that blocks commits if secrets are still decrypted

Already have a `.env`? vars will offer to import it during init.

## Add a variable

```bash
npx vars show
```

This decrypts the vault into `.vars/unlocked.vars`. Open that file and add a variable:

```vars
DATABASE_URL  z.string().url()
  @dev  = postgres://localhost/myapp
  @prod = postgres://prod-host/myapp
```

The schema (`z.string().url()`) sits right next to the name. vars validates against it, so a typo in the connection string gets caught before it reaches your app.

## Encrypt

```bash
npx vars hide
```

Encrypts everything back with AES-256-GCM and removes the unlocked file. Safe to commit.

## Run your app

```bash
npx vars run --env dev -- node server.js
```

vars decrypts in memory and injects the values as environment variables before handing off to your process. Nothing gets written to disk.

## Generate typed accessors

```bash
npx vars gen
```

Outputs `.vars/vars.generated.ts` with a typed export for each variable. No more guessing whether `process.env.DATABASE_URL` exists. This file is also auto-regenerated by `vars show`, `vars hide`, and the framework plugins.
